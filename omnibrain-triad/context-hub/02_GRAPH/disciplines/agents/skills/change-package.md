# Change Package

## Quando usar

Sempre antes de revisão; obrigatório em L3.

## Objetivo

Padronizar contexto para auditoria objetiva baseada em diff.

## Checklist

- preencher metadata;
- descrever contexto e critérios de aceitação;
- listar riscos;
- anexar links do Skill Graph;
- incluir `git diff`.

## Snippet mínimo

```text
Change-ID, Level, Goal, Acceptance Criteria, Git Diff
```

## Validação objetiva

- template completo sem seção vazia.
- diff anexado.

## Armadilhas

- descrição genérica sem critério verificável.
- diff truncado sem arquivo crítico.
