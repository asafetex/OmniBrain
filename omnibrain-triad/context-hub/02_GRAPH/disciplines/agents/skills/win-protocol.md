# WIN Protocol

## Quando usar

Após merge de mudança validada com utilidade reutilizável.

## Objetivo

Transformar resultado bom em ativo reutilizável do Graph.

## Checklist

- registrar `WIN` na memória;
- incluir evidência (Change-ID + resultado de gate/testes);
- extrair padrão reutilizável;
- promover para nó atômico quando passar curadoria.

## Snippet mínimo

```text
WIN = contexto + decisão + snippet + validação + armadilhas
```

## Validação objetiva

- WIN recuperável por tag/projeto.
- nó promovido com checklist e snippet.

## Armadilhas

- promover vitória sem prova.
- confundir resultado pontual com padrão.
