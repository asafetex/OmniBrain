# Memory Hygiene

## Objetivo

Manter memória útil, rastreável e compacta.

## Boas práticas

- registrar fatos e decisões, não opinião vaga;
- usar tags consistentes;
- adicionar referência de Change-ID;
- separar `WIN` de `LESSON`.

## O que evitar

- logs gigantes sem resumo;
- conteúdo duplicado;
- memória sem contexto mínimo (projeto, tópico, nível).

## Rotina semanal

1. revisar INBOX;
2. arquivar irrelevante;
3. promover wins reutilizáveis;
4. atualizar links quebrados no Graph.
