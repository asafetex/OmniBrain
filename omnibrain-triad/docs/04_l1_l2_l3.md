# 04 L1 L2 L3

## Definições

- L1 (baixo risco):
  - alteração local e reversível;
  - baixo impacto sistêmico.
- L2 (médio risco):
  - alteração funcional moderada;
  - pode exigir revisão externa.
- L3 (alto risco/crítico):
  - segurança, billing, schema/migration, pipeline crítico, core business.

## Definition of Done por nível

- L1:
  - diff pequeno e coerente;
  - build local básico passa (quando aplicável);
  - sem Gate obrigatório.
- L2:
  - critérios de aceitação cobertos;
  - testes locais relevantes;
  - Gate leve recomendado (`Codex` ou `Gemini`), PreGate opcional.
- L3:
  - Change Package completo;
  - Gate obrigatório com Codex + Gemini;
  - ambos precisam `VERDICT: APPROVE`;
  - testes e validações obrigatórios antes de commit.

## Quando rodar Gate/PreGate

- PreGate (DeepSeek/CodeRabbit):
  - opcional em L2/L3;
  - roda antes do Gate principal;
  - objetivo: capturar smells e melhorias rápidas.
- Gate principal:
  - L3: obrigatório;
  - L2: recomendado quando houver risco não trivial;
  - L1: normalmente não necessário.
